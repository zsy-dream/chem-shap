# 01 - 项目总览

## 项目名称
**智析实验——基于SHAP值的化学实验条件归因优化平台**

## 项目定位
用可解释AI技术帮助化学实验研究者理解"为什么这组条件好/不好"，把实验优化从"经验试错"升级为"数据驱动+可解释决策"。

## 赛道与方向
- **参赛赛道**：高教主赛道
- **技术方向**：新材料/绿色化工 + 人工智能
- **项目阶段**：MVP 已完成，具备基本演示能力

## 一句话价值主张
让实验结果"说得清楚"，让下一轮实验"调得明白"。

## 核心痛点
化学实验中的真实问题：
- 一组实验做得不好，到底怪温度、催化剂、pH，还是配比？
- 同样的反应体系，换一组参数结果差很多，不知道关键因素在哪
- 实验记录散落在Excel和笔记本里，难以系统分析和复用
- 机器学习能预测成功率，但研究者还是不知道"为什么"和"下一步怎么调"

## 解决方案闭环
```
上传实验数据（CSV）→ 训练预测模型 → 选择实验样本 → SHAP归因分析 → 查看贡献排序 → 获取调参建议 → 生成分析报告
```

## 已实现能力
- **Web界面**：基于Flask的简单Web操作界面
- **样本管理**：支持实验样本录入、分组、历史记录查看
- **模型训练**：支持XGBoost/RandomForest/LightGBM三种模型训练
- **模型评估**：AUC、F1、Accuracy等基础评估指标
- **SHAP分析**：个体样本解释、全局特征重要性分析
- **可视化图表**：瀑布图、力导向图、摘要图、依赖图（Base64输出）
- **建议生成**：基于Top特征贡献的简单调参建议
- **报告系统**：分析结果存储、查询、展示（每次分析形成可追溯记录）
- **基础API**：数据上传、模型训练、SHAP分析等RESTful接口
- **Docker部署**：支持容器化部署

## 数据与存储方式（贴近实现）

系统当前采用 SQLite + 文件存储组合：

- **实验记录**：每条实验记录以 JSON（`feature_data`）形式写入数据库，便于灵活扩展字段
- **模型文件**：训练好的模型以文件（如 `.pkl`）形式保存，并在数据库记录元信息（名称、版本、指标、是否激活）
- **分析报告**：将达标概率、SHAP贡献值、Top特征与建议文本写入报告表，便于复盘与对比

## 实验数据字段
当前支持的8个化学实验参数：
- reaction_temperature（反应温度，℃）
- reaction_time_min（反应时间，分钟）
- ph_value（pH值）
- catalyst_loading（催化剂添加量，%）
- solvent_polarity（溶剂极性，0-10）
- stirring_speed_rpm（搅拌转速，rpm）
- reactant_ratio（反应物配比）
- crystallization_time_min（结晶时间，分钟）
- target（目标变量：0=待优化，1=达标）

## 可演示成果（比赛现场能讲清楚）

- **演示数据**：仓库提供 `sample_data.csv` / `sample_data_large.csv` 作为快速测试与训练样例
- **演示流程**：上传数据→训练模型→激活模型→选择样本→一键SHAP分析→生成图表与报告
- **解释输出**：瀑布图/力导向图等图表由后端生成并转 Base64，可直接展示与截图用于答辩材料

## 项目特色
- **解释性强**：基于SHAP算法，量化每个实验条件对结果的贡献值
- **操作简单**：Web界面操作，不需要编程基础
- **结果直观**：图表化展示，适合科研汇报
- **成本可控**：基于开源技术栈，部署成本低
- **可扩展**：模块化设计，便于后续功能扩展

## MVP边界说明（不夸张、但有方向）

- 当前版本更适合做教学/科研工具原型与试点验证：强调“可解释”和“可复盘”
- 对企业级大规模部署（高并发、审计合规、内网安全体系）目前只做了基础设计与接口预留，后续将根据真实合作需求再增强
